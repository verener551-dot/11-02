# Домашнее задание к занятию "`Базы данных, их типы`" - `Шалыгин Денис`

### Задание 1

Кейс
Крупная строительная компания, которая также занимается проектированием и девелопментом, решила создать правильную архитектуру для работы с данными. Ниже представлены задачи, которые необходимо решить для каждой предметной области.

Какие типы СУБД, на ваш взгляд, лучше всего подойдут для решения этих задач и почему?

1.1. Бюджетирование проектов с дальнейшим формированием финансовых аналитических отчётов и прогнозирования рисков. СУБД должна гарантировать целостность и чёткую структуру данных.

1.2. Под каждый девелоперский проект создаётся отдельный лендинг, и все данные по лидам стекаются в CRM к маркетологам и менеджерам по продажам. Какой тип СУБД лучше использовать для лендингов и для CRM? СУБД должны быть гибкими и быстрыми.

1.3. Отдел контроля качества решил создать базу по корпоративным нормам и правилам, обучающему материалу и так далее, сформированную согласно структуре компании. СУБД должна иметь простую и понятную структуру.

1.4. Департамент логистики нуждается в решении задач по быстрому формированию маршрутов доставки материалов по объектам и распределению курьеров по маршрутам с доставкой документов. СУБД должна уметь быстро работать со связями.;%

### Ответ

1. Реляционная СУБД - лучше подходят для задачь требующих целостности данных структуры и сложных запросов с агрегацией. Они соблюдают ACID-принципы что важно для финансовых операций и отчётности.
2. Реляционные СУБД - гарантирует целостность и структуру данных о клиентах сделках и взаимодействиях.
3. Графовая СУБД - позволяет моделировать и запрашивать связи показывая все документы относящиеся к конкретному отделу и всем его подразделениям.
4. Графовая СУБД - созданы для быстрого анализа связей и решения таких задач как поиск кратчайшего пути или оптимального маршрута для нескольких курьеров.

### Задание 2

2.1. Пользователь пополняет баланс счёта телефона, распишите пошагово, какие действия должны произойти для того, чтобы транзакция завершилась успешно. Ориентируйтесь на шесть действий.

### Ответ

2.1. Инициация транзакции - Пользователь вводит номер телефона и сумму пополнения, формируеться запрос на проведение платежа.
2.2. Проверка данных и доступности сервисов - корректность номера телефона, доступность платёжного шлюза и биллинговой системы оператора для зачисления средств.
2.3. Авторизация платежа - банк подтверждает возможность списания и резервирует средства.
2.4 Списание средств и зачисление. 
2.5 Подтверждение успешного выполнения.
2.6 Фиксация результата - все системы участвовавшие в процессе, сохраняют в своих логах полную информацию о транзакции: время, сумма, номер телефона, идентификатор платежа.

### Задание 3

3.1. Напишите пять преимуществ SQL-систем по отношению к NoSQL.

### Ответ

3.1. ACID-транзакции. 
SQL-базы данных обычно используют ACID-комплект свойств, что означает, что транзакции выполняются совместно и что любые откатывания транзакций произойдут для всех соединений с данными. Это значит, что если транзакция завершится неудачей, все изменения будут отменены и база данных будет возвращена к предыдущему состоянию.
NoSQL-базы данных, такие как MongoDB или Cassandra, обычно не используют транзакции, так как они нацелены на большие масштабы данных и скорость обработки, нежели на поддержку обеспечения транзакционности.
3.2. Гибкость и мощность запросов - SQL является стандартизированным и универсальным языком, который позволяет выполнять сложные запросы, объединять данные из нескольких таблиц и проводить глубокую аналитику.
3.3. Широкая совместимость - SQL-базы данных совместимы с широким спектром инструментов, приложений и языков программирования, что упрощает интеграцию и снижает затраты на разработку.
3.4. Структурированные данные - SQL-системы идеально подходят для работы с хорошо структурированными данными, где связи между таблицами четко определены.
3.5. Мощные транзакции - Они обеспечивают полную поддержку транзакций, что позволяет выполнять сложные операции, которые требуют атомарности (все или ничего), в отличие от NoSQL, где поддержка транзакций может быть ограничена.

### Задание 4

Необходимо производить большое количество вычислений при работе с огромным количеством данных, под эту задачу выделено 1000 машин.

На основе какого критерия будете выбирать тип СУБД и какая модель распределённых вычислений здесь справится лучше всего и почему?

### Ответ
Критерий
Горизонтальная масштабируемость - Система должна равномерно распределять нагрузку на все узлы кластера. Для этого лучше подходит NoSQL СУБД
Поддержка распределенных транзакций - Обеспечение согласованности данных across множества узлов - СУБД с поддержкой ACID в распределенной среде
Тип данных и модель - Зависит от структуры данных - структурированные/неструктурированные - Колоночные СУБД для аналитики, документные для гибких данных
Отказоустойчивость - При 1000 машинах failures неизбежны - нужна репликация - Системы с автоматической репликацией и восстановлением
Больше всего подходт NoSQL Column-oriented
